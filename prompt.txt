DATABASE (better-sqlite3, auto-creates on startup):
Tables:
- users: id, email, password_hash, role, created_at
- products: id, name, category, price, description, stock, sizes(JSON string), image_url, created_at
- orders: id, customer_name, phone, email, address_line1, address_line2, city, state, pincode, items(JSON string), total, status(default Pending), created_at

API ROUTES:
Public:
  GET  /api/products (query params: category, search)
  GET  /api/products/:id
  POST /api/orders (place COD order)

Auth:
  POST /api/auth/login
  POST /api/auth/logout
  GET  /api/auth/me

Admin (JWT protected, admin role only):
  GET    /api/admin/stats
  GET    /api/admin/products
  POST   /api/admin/products (multipart/form-data with image)
  PUT    /api/admin/products/:id (multipart/form-data with image)
  DELETE /api/admin/products/:id
  GET    /api/admin/orders
  PUT    /api/admin/orders/:id (update status only)

SECURITY:
- bcryptjs for passwords (saltRounds: 12)
- JWT in httpOnly cookies (maxAge: 24h)
- helmet.js for headers
- express-rate-limit on login (10 req/15min)
- express-validator on all POST/PUT routes
- CORS: allow only localhost:5173 in dev
- Multer: only jpg/png/webp, max 5MB, rename to uuid
- Parameterized queries only, no string concat in SQL

CART: stored in localStorage, managed by CartContext
No cart table needed in DB.

CHECKOUT: COD only. Customer fills:
Full Name, Phone, Email, Address Line 1, Address Line 2, City, State, Pincode
On submit: POST /api/orders → save to DB → redirect to /order-success → clear cart

ADMIN FEATURES:
- Login at /admin/login with email+password
- Dashboard: stats cards (total products, orders, pending, revenue)
- Products: list, add (with image upload), edit, delete
- Orders: table with customer details, items, total, status
- Update order status: Pending → Shipped → Delivered
- Image upload: stored in server/uploads/, served as static

IMAGE UPLOAD FIX:
- Frontend sends FormData with image file
- Backend multer saves to server/uploads/
- Response includes image_url as /uploads/filename.ext
- Frontend displays as http://localhost:5000/uploads/filename.ext
- In vite.config.js proxy /uploads to http://localhost:5000

STOCK CONTROL:
- CartContext: if item quantity >= product.stock, block add and show message
- ProductDetailPage: quantity max = product.stock, show "Only X left" if stock <= 5
- Show "Out of Stock" and disable button if stock = 0
